# ADR-0003：詳解三段固定結構＋合併解析（棄自由散文）

- **狀態**：已採用（Accepted，2026-07-16）
- **日期**：2026-07-16
- **決策者**：專案維護者
- **相關**：`references/explanations-redteam.md` §1.4／§1.5（本決策的操作層規則）；ADR-0005（token 工程，三段結構是零 token 結構 lint 得以成立的前提）；ADR-0006（模型分層）；ADR-0008（三層防錯，結構 lint 是其地基之一）。

## 背景（Context）

詳解要跨多科、數萬則量產。最初的生成通則只有一句「先點正解、再逐一辨析其餘選項」。隨機抽每科數十則實測發現，這種自由散文寫多了會系統性走鐘：

- 選項辨析黏成一坨，考生分不出哪句在講哪個選項；
- 考點常整段漏講；
- 前端渲染不出段落感（`.explain-body` 是 `white-space:pre-line`，沒有 `\n\n` 就擠成一片）；
- 更關鍵的：自由散文**無法被零 token 腳本驗證**——沒有固定錨點，lint 抓不到「有沒有逐項辨析」「考點有沒有寫」，品質只能靠昂貴的模型逐則覆核，成本倒置。

用「考生視角健檢」（讓模型扮演實際讀詳解的考生、模擬其閱讀體驗）＋實測驗證，收斂出：詳解的價值不在文采，而在**可掃描、可重算、可對照選項字母**，這要求固定的結構契約，不是一句通則。

## 決策（Decision）

**每則詳解固定三段，段間以空行（連續兩個換行 `\n\n`）分隔：**

1. **正解段（單獨自成一段）**：以樸素標記「正解：」起頭，先明講哪個／哪些選項正確與為什麼；複選題逐一講每個正確選項；計算題的逐步演算放這段。
2. **選項辨析段（每個非正解選項各自成一段）**：每段以該選項大寫字母＋全形冒號起頭（「A：」「B：」），逐一講為何錯。深度依題型分流——純數字／單一事實記憶題講清依據即可、可低於 120 字；誘答本身是實質概念（易混淆的學說、方法、分類）時必須點出關鍵差異並說明「為什麼這個誤答看起來對」。
3. **本題考點段（末段、前空一行）**：以「本題考點：」起頭，列 1–3 個概念／理論／專有名詞（有通用英文原名的附上），一句話點出它是什麼。

**配套硬規則：**

- **選項指稱一律用 A–E 大寫字母**，禁用試題本原始數字編號（考生畫面看到的是字母標記，詳解用數字會對不上）。
- **計算題必逐步演算**、不受一般字數上限，以「考生可重算一次」為準；每個誘答都要算出它錯在哪一步／哪個數字，不得以「基礎不同故不符」一語帶過。概念辨析型以 600 字為度。
- **正文須為乾淨成品**：不得夾帶修正註記、審查對話、自我驗證旁白（「（更正…）」「經查…」這類鷹架）。法規題應引用現行法條號或官方公開命題／評分說明當查證錨點（公共領域、非受保護編排）。
- **合併解析（單選、複選一體適用）**：僅當兩個以上選項的判斷理由**本質相同、分開寫必然逐句重複**時，合併為同一段，段首以該組字母依選項順序、頓號「、」相連再加全形冒號起頭（「A、C：」）。門檻嚴格限縮為「同一判準／同一條法規／同一個概念錯位」的同因情形；合併後仍須逐一點出各選項字面差異。理由不同、僅結論同為錯者一律各自分段；計算題各誘答錯在不同步驟或數字屬本質不同，不得合併。

一則符合本結構的完整範例，見 [`examples/pipeline-templates.md`](../../examples/pipeline-templates.md) 的「金樣詳解範例」。

## 考慮過的替代方案（Alternatives）

- **A. 維持自由散文通則**（只給「先講正解、再辨析」）：實作最簡單，但實測既有大量舊詳解無一可被結構 lint 驗證，品質只能靠模型逐則覆核，成本倒置；且前端渲染不出段落感、考生難掃描。已被本 ADR 取代——正是這批舊詳解全部需要重構（見 ADR-0004）。
- **B. 摘要／抽樣式辨析**（只講最關鍵的一兩個誘答）：省版面，但賭「哪個誘答不重要」，且對複選與概念混淆題會漏掉考生真正卡的選項。
- **C. 另設「難度」／「常錯」欄位**標記高頻陷阱：資訊更結構化，但**過度規範**——維護成本高、標記本身會引入主觀判斷與錯誤。改為「若某誘答是歷年高頻陷阱，直接在辨析句裡點明」，不另立欄位。
- **D. 用 markdown 標記段落**（`##`、`-`）：前端不渲染 markdown，且 markdown 符號會污染 `white-space:pre-line` 的純文字渲染。改用樸素中文標記（「正解：」「本題考點：」）＋空行。

## 後果（Consequences）

- 三段結構讓**零 token 結構 lint 成立**：首段／末段標記、每個非正解字母恰出現一次、段落空行分隔、計算題含「數字＝數字」步驟——全可純腳本驗，結構有問題的則直接退回重生、不浪費紅隊額度。這是 ADR-0005 token 工程與 ADR-0008 三層防錯的地基。
- 詳解可掃描、可對照選項字母、計算題可重算，考生體驗一致。
- 代價：三段完整優先於字數通則，部分則會變長（尤其計算題與多誘答概念題）；生成模型需要更嚴格的模板遵循（靠 ADR-0008 的金樣 few-shot＋蒸餾操作卡拉齊）。
- 「合併解析」的同因門檻是主觀判斷點，過寬會讓考生分不出選項、過嚴會逼出逐句重複的多段——實測以「同一判準／同一條法規／同一概念錯位」三種情形為界。

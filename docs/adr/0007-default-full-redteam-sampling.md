# ADR-0007：紅隊預設 100% 全抽、降抽要掙（正確性＞成本）

- **狀態**：已採用（Accepted，2026-07-18）
- **日期**：2026-07-18
- **決策者**：專案維護者
- **相關**：`references/explanations-redteam.md` §2.3（預設全抽原則）、§3（紅隊抽樣）；ADR-0004（重構線淨新增內容）；ADR-0005（分層 schema 壓低全抽代價）；ADR-0008（三層防錯＝降抽的前提條件）。

## 背景（Context）

紅隊抽樣率是「正確性 vs 成本」的直接槓桿。早前規範傾向「抽樣＋統計停止規則」（Wilson CI 下界動態停），把抽查當預設、全跑當例外。一次真實戰役翻轉了這個預設：

**某國考社工師科的重構線首波實測**——重構後只抽審了約 **24%**，就揪出約 **10%** 的真錯率，錯誤集中在補寫的法律事實層（法條號張冠李戴、法定數字錯如 7 日→10 日、虛構規定、權責用語錯如偵查／調查）。

關鍵推論：10% 真錯率下只抽 24%，意味著未抽的部分可能藏著**數量可觀的錯直接上線**。抽樣的前提是「首過品質已被實測證明夠好」——當首過品質實測不夠好時，抽查等於放行未知數量的錯。學習站的價值排序是「正確性對使用者＞成本」，抽樣預設必須翻轉。

## 決策（Decision）

**每科×每線預設 100% 全抽。** 正確性對使用者的價值高於成本；分層 schema（`ok` 零 reason，見 ADR-0005 槓桿 4）已把全抽代價壓掉大半，別拿舊成本想像嚇自己。

**降到部分抽樣的條件（全部成立才可降）：**

1. 該科零 token 三層已生效（lint 黑名單＋內容守恆檢查＋法條引用核對器）；
2. **連續兩批實測真錯率 ≤2–3%**（高危科門檻減半）。

**回升機制**：降抽後任一批回升破門檻→該科**立即回 100%**，修好 prompt／金樣後重新掙。

**降抽後仍守的下限**：已降抽的科走**風險加權**——低把握度題與法條核對器 flag 的題**永遠 100%**，其餘走 golden-batch 定的基準比。

**存量補救（優先於新批次）**：已入庫但未抽審的存量，在法條引用核對器成品後先跑**全量零 token 掃描**（flag 全審），再依預設全抽政策補足紅隊；掃描不必等生成 loop 啟動，核對器完成當天即可跑。

## 考慮過的替代方案（Alternatives）

- **A. 抽查為預設＋Wilson CI 動態停**（早前版本）：成本最低，統計上「錯誤率確實偏高才升級」。但實測證明**前提「首過品質夠好」不成立**——10% 真錯率下抽 24% 等於放行數量可觀、已知型態的錯。前提不成立時，動態停止規則保護的是成本不是使用者。已被翻轉為「全抽為預設、抽查為掙來的例外」。
- **B. 固定百分比抽樣（如各科 20–40%）**：比 A 保守，但仍是「賭未抽部分乾淨」，且固定比例不隨首過品質調整。改為「預設全抽、降抽要連續兩批達標掙得」，把抽樣率綁到實測品質而非拍板比例。
- **C. 全抽但取消低把握度／flag 的 100% 下限**（降抽後一視同仁）：簡單，但低把握度與核對器 flag 恰是最可能藏錯的子集，取消下限等於在最危險處省力。保留為永遠 100%。

## 後果（Consequences）

- **正確性優先落地**：預設全抽杜絕「未抽部分藏數量可觀的錯」的系統性風險；分層 schema 讓全抽代價可負擔。
- 代價：紅隊 token 量比抽查高（但被 ok 零 reason 壓掉大半，且被 ADR-0005 省下的重複稅回補）。
- **抽樣率與輸出格式分開調**（ADR-0005 總原則二）：全抽是抽樣率（品質參數）、分層 schema 是輸出格式（成本參數），兩者不互相綁架——全抽不必然貴。
- 降抽是「實測掙來的例外」，需要零 token 三層先生效＋連續兩批達標，且隨時可能因一批回升被打回全抽——這把「降抽」從預設變成需要持續維持的資格。

## 如何重啟討論／修改（How to revisit）

- 預設全抽是價值排序的裁定（正確性＞成本）。若未來價值排序改變（如純成本受限的重跑），需重新裁定，不由執行端自行降預設。
- 降抽門檻（≤2–3%、高危科減半）與風險加權下限（低把握度／flag 永遠 100%）是實測校準值，隨 golden-batch 與各科真錯率統計更新；本 ADR 記的是「為何全抽為預設」的決策邏輯與實測依據。

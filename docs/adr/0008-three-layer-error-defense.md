# ADR-0008：三層防錯分工＋金樣先過紅隊＋fresh-context 鐵律

- **狀態**：已採用（Accepted，2026-07-18）
- **日期**：2026-07-18
- **決策者**：專案維護者
- **相關**：`references/explanations-redteam.md` §2.1（品質拉齊五機制）、§2.3（三層分工）、§3（紅隊）；ADR-0003（三段結構＝結構 lint 的地基）；ADR-0005（token 工程）；ADR-0006（模型分層）；ADR-0007（抽樣率）。

## 背景（Context）

量產詳解的模型（下稱**執行模型**）品質難免比直接下筆的**強模型**鬆散。若只靠事後紅隊一層兜底，紅隊要同時抓格式錯、事實錯、教學鬆散，火力被稀釋、成本爆表，而且結構有問題的則送進紅隊只是浪費一次查核額度。需要一套**縱深防禦**：讓便宜的零 token 手段先擋掉它們擋得住的，把昂貴的紅隊火力集中在只有它能抓的事實錯。

同時，兩個容易被忽略的洞決定這套防禦能不能成立：**範本（金樣）自己若埋錯會系統性汙染**，以及**紅隊隔離的對象到底是什麼**。

## 決策（Decision）

**一、三層防錯分工（各司其職、不重疊）：**

1. **結構 lint 抓格式（零 token）**：純腳本檢查三段標記齊全、每個非正解字母恰出現一次、段落空行、選項只用 A–E、無鷹架詞、計算題含「數字＝數字」步驟、字數下限。任一項不過就**整則退回重生、不送紅隊**。這一層的成立仰賴 ADR-0003 的固定結構契約。
2. **檢索錨定防事實錯**：prompt 隨題附上該科現行法條／公式表／課綱片段，讓執行模型「抄現行條文」而非「憑記憶覆誦」，從源頭壓低杜撰法條號、引用舊法規的機率。法規科更進一步用**法條引用核對器**（零 token）抽詳解裡的「法名＋條號＋關鍵數字」去比對素材包全文，虛構條號直接攔、關鍵數字對不上就 flag。
3. **紅隊抓剩下的事實錯**：反駁式（不是覆述式）由強模型主動找碴——正解寫反？措辭誤導？事實過時？把握度標太高？前兩層零 token 手段抓不到的、需要「知道對不對」的事實錯，全靠這一層。

**二、金樣先過紅隊（範本埋錯會系統性汙染）：** 用作 few-shot 的金樣詳解，**本身必須先過紅隊才可入 prompt**。範本自己若埋錯，會系統性汙染同主題的所有生成——實測有一則金樣把某法條號寫錯，連累了同主題的數則生成。未查核的金樣是放大器，不是護欄。一則達到金樣水準（三段結構、逐選項辨析、考點段、自足不依賴附圖）的完整範例，見 [`examples/pipeline-templates.md`](../../examples/pipeline-templates.md) 的「金樣詳解範例」。

**三、fresh-context 鐵律（隔離撰寫者、不是隔離同批題）：** 紅隊必須在與撰寫者隔離的 fresh context 進行，才不會「附和」撰寫者的錯。但**隔離的對象是「撰寫者思路」，不是「同批其他題」**——題目彼此獨立時，同一紅隊 call 批 10–20 題不損隔離，還大省殼開銷（呼應 ADR-0005 槓桿 4）。把 fresh-context 誤解成「一題一 agent」會白白燒掉數十萬 token 而無額外保護。

**四、品質拉齊五機制（把執行模型拉近強模型水準）：** 金樣 few-shot、結構 lint（零 token 硬門）、檢索錨定、分級路由（把握度偏低／法規時效敏感／官方曾釋疑的題一開始就交強模型）、審計回饋圈（每批把發現的錯誤模式歸納成「常見雷」回灌下一批）。與其事後靠紅隊逐則挑鬆散，不如把拉齊內建進生成流程。詳見 `references/explanations-redteam.md` §2.1。

**五、摻毒批 recall 鐵律：** 改紅隊 batch size 或換模型層級時，**必重跑摻毒批召回率**——在試點批植入已知錯誤（法條號改錯、法定數字改錯、結論講反各數則），量紅隊抓回幾個。否則 schema 化／批次化省下的 token 可能是拿偵測率換的而不自知（真實批次沒有 ground truth，摻毒是唯一能量召回率的方法）。

## 考慮過的替代方案（Alternatives）

- **A. 只靠紅隊一層（不分工）**：紅隊要抓所有錯，格式錯浪費查核額度、成本爆表，且結構壞的則佔用本該給事實查核的火力。改為三層縱深，零 token 手段先擋格式與可核對的事實。
- **B. 金樣不先過紅隊（快速起步）**：省一道前置，但範本埋錯會系統性汙染同主題的所有生成——放大器，不是護欄。已排除。
- **C. 紅隊一題一 agent 求「最純 fresh context」**：誤解 fresh-context——隔離對象是撰寫者思路，不是同批題。題目獨立時批次同審不損隔離、大省殼開銷。
- **D. 靠模型自審取代獨立紅隊**：自審容易附和自己的錯，覆述式尤其如此。改用與撰寫者隔離的**反駁式**獨立紅隊，才逼得出真錯。

## 後果（Consequences）

- 三層互補、火力集中：便宜手段擋格式與可核對事實，紅隊只打它能打的，整體召回率高、成本低。
- 金樣過紅隊是一道前置成本，但擋掉的是會被 few-shot 放大的系統性汙染，划算。
- fresh-context＋批次化兼顧隔離與省 token，是 ADR-0005 與本 ADR 的交會點。
- 代價：三層都要維護（lint 檢查項、素材包與核對器、紅隊 schema 與摻毒批），任一層失效都會讓縱深出現缺口——摻毒批 recall 就是持續驗證缺口有沒有出現的手段。

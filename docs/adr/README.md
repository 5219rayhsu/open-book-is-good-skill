# 架構決策紀錄（ADR）— 方法論公開版

這個資料夾收「開卷有益 Open-Book-Is-Good」復刻手冊背後的**方法論決策紀錄**（Architecture
Decision Records）。每一篇記一個影響整套管線的技術取捨：為什麼這樣做、考慮過哪些替代方案、
帶來什麼後果、日後怎麼重啟討論。

## 這裡收什麼、不收什麼

- **收**：可被任何人復刻／共編所需要的**方法論取捨**——詳解結構、重構策略、token 工程、
  模型分層、紅隊抽樣、三層防錯。這些是「怎麼做、為什麼這樣做」的通用工程知識。
- **不收**：實際的產物內容與內部運營狀態——各科題數與合規率、科目優先序、逐檔生產路徑、
  排程作戰計畫。ADR 記的是決策邏輯，不是生產快照；那些會隨題庫變動，也不是共編需要的東西。

一句話：**ADR 是決策層（為什麼），`SKILL.md` 與 `references/` 是操作層（怎麼做）。** 兩者互補，
改任一層的規則前，先讀對應的 ADR 理解它為何長這樣。

## 格式

每篇 ADR 固定欄位：**狀態**、**日期**、**決策者**、**相關**，接著 **背景（Context）**、
**決策（Decision）**、**考慮過的替代方案（Alternatives）**、**後果（Consequences）**、
**如何重啟討論／修改（How to revisit）**。繁體中文（臺灣）、全形標點。

想動方法論規則、或對某個取捨有不同看法，歡迎開一個 GitHub Issue 連回對應 ADR 討論
（見 [CONTRIBUTING.md](../../CONTRIBUTING.md)）。

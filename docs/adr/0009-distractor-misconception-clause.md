# ADR-0009：概念型誘答的「為什麼它看起來對」硬性條款

- **狀態**：已採用（Adopted）
- **日期**：2026-07-22

## 決策

概念型誘答（易與正解混淆的學說／方法／分類）每個至少一句「為什麼它看起來對」的誘答設計說明；
純數字／年份替代型誘答免。落點：操作卡＋生成 prompt＋紅隊「辨析是否成立」檢查一併驗證此句
存在與合理。不加結構 lint（屬語意判斷）、不設獨立欄位（避免過度規範）。

## 原因

詳解的回饋可分四層：對錯（outcome）、錯在哪（informational）、正確為何（corrective）、
**為什麼這個誤答看起來對（misconception repair）**。前三層由結構 lint 與紅隊已結構性保證；
第四層缺席時，考生答錯背後的誤解未被修正，同一誤解會在相似題重複觸發——出題者的誘答
本來就是針對常見誤解設計的，詳解只證明「它錯」而不解釋「它為何誘人」，等於放過了
題目最有教學價值的部分。

同時不採「全面加厚詳解」：短而結構穩定的判準句是更好的記憶提取線索，回饋的閱讀成本
會直接吃掉練習量——難度應該留在練習裡，不在回饋裡。因此只補一句、不增字數要求。

## 後果

- 精瘦骨架保留，僅概念型誘答補一句教學核心。
- 金樣更新時同步示範此句寫法。
- 既有批次不回溯重跑，由後續紅隊輪抽到再補。

## 補遺（2026-07-23 收斂）

大批量實測顯示：法規科幾乎所有錯誤選項都屬概念型，「逐選項一句誘答說明」的判準會讓紅隊 fix 率失真飆高、詳解冗長。條款收斂為：每題只對「最強誘答」1–2 個寫誘答說明；同因合併段可用一句判準句（「分辨關鍵在 X」）取代；紅隊僅當「最強誘答既無誘答說明也無判準句」才記低嚴重度。

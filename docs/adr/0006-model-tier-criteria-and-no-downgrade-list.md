# ADR-0006：模型分層三級判準＋不降清單（含醫學科官方限制）

- **狀態**：已採用（Accepted，2026-07-18）
- **日期**：2026-07-18
- **決策者**：專案維護者
- **相關**：`references/explanations-redteam.md` §2.3.1（本決策的操作層規則）；ADR-0005（token 工程）；ADR-0007（抽樣率）。

## 背景（Context）

ADR-0005 的七槓桿把「省重複稅」對齊「升品質」，但留下一個未收斂的問題：**哪個環節該用哪級模型？** 沒有可判準則會出兩種誤配——把貴模型燒在腳本能做的事（浪費），或把便宜模型放進沒有兜底驗證網的環節（未審事實直通入庫）。需要把這件事收成一條可判的準則並鎖死，防未來 token 工程壓力下誤降。

另有一個**官方限制**需要落成規則：醫學／臨床科內容涉及醫療判斷，受模型供應商對醫療建議的使用規範約束——即使主控是最高層級模型，這類內容的處理實務上會落到最強模型（Opus）。這**不是**品質設計選擇，是當前官方限制。

## 決策（Decision）

**三級判準：**

- **腳本能證明的不花 token**：確定性映射、集合比對、規則查表一律零 token 腳本先行。
- **輸出可被零 token 腳本完整驗證的環節才可降便宜模型（Haiku）**：格式修、樣式辨識這類「錯了 lint／守恆檢查抓得到」的活。
- **需要模型「知道對不對」（憑空寫事實）的環節不降**：生成事實、裁決真偽，一律維持中階以上。
- **鐵律**：任何降級規格必須寫明兜底驗證網是哪道門，寫不出即不准降。

**Python 先行原則**：能被前兩層零 token 手段接手的，一律先寫成腳本，剩下的才交模型（例：送分題＝答案標記查表；官方曾釋疑＝清單查表；選項圈號塌陷＝造字碼位對照表的確定性映射，先於任何 OCR／看圖——**但對照表是 per-font 的**，跨卷套用前先驗字型出處）。連錯誤類別統計、誤報率都走腳本 tally，強模型只留「把統計出的雷歸納成可回灌 prompt 的判斷」那半邊。

**機器處理過≠免抽**：當「驗證」與「轉換」共用同一套啟發式（如選項編號轉字母後用 lint 驗字母覆蓋），那道驗證是**警告不是證明**——經自動轉換的題照常留在紅隊抽樣池，不因機器處理過而豁免。

**不降清單（審過鎖死，防未來誤降）**：紅隊修改中「自由文字方向的大改」（連帶改動風險無便宜機器可驗）、圖表看圖複判（只開放小樣本 A/B、不直接降）、紅隊本體、新寫線的生成主幹、裁決者、金樣撰寫、申論解析、以及高風險科（如醫學）的**裁決／覆核與高危題型撰寫**（撰寫主幹本身可經 A/B 實測下放，見下方醫學科段）——凡「模型必須知道對不對」的環節一律鎖死。便宜模型只在小樣本 A/B 實測勝出、且該環節有零 token 兜底網時才下放。

**醫學／臨床科：官方限制綁「把關」、撰寫是品質問題（安全提醒）：**

- **裁決／覆核落最強模型（Opus）＝官方限制**：醫療建議的把關受模型供應商使用規範約束，這是官方限制、非品質選擇，不因成本下放。
- **撰寫是品質問題、可經 A/B 下放**：醫學科撰寫**經 golden-batch A/B 實測（雙軌對比殘餘真錯率＋摻毒 recall 校準）可下放中階模型（Sonnet）**——實測顯示中階模型的醫學撰寫品質與最強模型無可測差異時，即可下放，省數倍成本。這正是把「官方限制（把關）」與「品質選擇（撰寫 tier）」分開的具體落地。
- **撰寫下放後的配套（缺一不可）**：① 高危題型（用藥劑量／急救處置／禁忌症）以關鍵詞表路由最強模型撰寫；② 出口 100% 全抽、全抽門檻不減；③ 低把握度題 100% 由最強模型覆核；④ 講反類（推理方向反、方位詞反）由零 token 結論一致性檢查前置補強——紅隊對講反類召回率相對弱，機器補位。
- **給復刻者的提醒**：撰寫 tier 一定要自己跑 A/B＋摻毒 recall 實測才下放，別直接套用「別人下放了所以我也下放」；裁決／覆核與高危題型撰寫的最強模型下限不可省；供應商醫療政策變動時全部重審。

**A/B 雙軌與摻毒 recall 的適用範圍：**

- **摻毒 recall＝每科預設校準**：每科 golden-batch 一律植入若干已知錯誤（講反／法定數字／劑量閾值／虛構機轉別名／竄改診斷準則，隱蔽而非明顯），與乾淨題混批交 fresh 紅隊，量**召回率**——沒有它，「真錯率低」可能只是紅隊盲、數字是空的。改紅隊 batch size 或換 tier 時必重跑。
- **A/B 雙軌 ≠ 每科預設**：雙軌（同批同金樣、兩模型對比殘餘真錯率）只回答「能否從貴模型**降**到便宜模型」。全站主幹本就是便宜模型，多數科不存在這個問題——每科都跑雙軌是拿兩倍生成成本回答一個對多數科不存在的問題。**只有「預設用最強模型的高風險科」才跑 A/B**（如受官方醫療規範限制的醫學／臨床科），其餘科走單軌 golden-batch＋摻毒 recall 即可。

## 考慮過的替代方案（Alternatives）

- **A. 全程最強模型**（生成、審查、修復都用 Opus）：品質上限最高，但把貴模型燒在腳本能做的事，千批規模成本爆表，且違反 ADR-0005「Python 先行」。
- **B. 全程便宜模型＋事後紅隊兜底**：成本最低，但「生成事實」「裁決真偽」沒有零 token 兜底網，紅隊抽樣一旦降就漏，未審事實直通入庫。這正是三級判準第三條要擋的。
- **C. 醫學科全科一律最強模型撰寫**：對醫學最安全，但把「品質選擇」與「官方限制」混為一談，也可能過度保守（部分基礎醫學題中階模型足夠）。改為「出口從嚴＋撰寫 tier 由 A/B 實測定＋高危題型關鍵詞路由最強模型」，把官方限制（覆核落 Opus）與品質選擇（撰寫 tier）分開，各自用對的機制。
- **D. 「機器處理過就免抽」**：省抽樣，但轉換與驗證共用同套啟發式時，那道驗證是警告不是證明——會漏掉啟發式本身的系統性錯。已明文排除。

## 後果（Consequences）

- 每個環節的模型 tier 有可判準則與兜底網，防未來 token 壓力下誤降；不降清單鎖死高風險環節。
- 「可降 Haiku」的每一項都綁定一道零 token 兜底門，兜底門失效（如守恆檢查誤殺率過高）時該降級即失去正當性，須回退。
- 代價：撰寫 tier 需要 golden-batch A/B 實測才定，不能拍腦袋；高危題型關鍵詞表需維護。

## 如何重啟討論／修改（How to revisit）

- 不降清單是審過鎖死的：任何新增降級必須先寫出兜底驗證網、經審計，再修訂 `references/explanations-redteam.md` §2.3.1。
- 醫學科的官方限制隨模型供應商政策變動——若供應商放寬醫療內容規範，本節例外須重審；在此之前一律從嚴。

# ADR-0005：詳解管線 token 工程七槓桿（省 token＝升品質）

- **狀態**：已採用（Accepted，2026-07-17）
- **日期**：2026-07-17
- **決策者**：專案維護者
- **相關**：`references/explanations-redteam.md` §2.3（本決策的操作層規則）；ADR-0003（三段結構＝結構 lint 的前提）；ADR-0006（模型分層）；ADR-0007（抽樣率）；ADR-0008（三層防錯＋金樣先過紅隊）。

## 背景（Context）

當題庫大到「數萬則詳解、數百批、跨越額度重置的多日背景工程」，總成本不再由「單則詳解生成」主宰，而由兩個隱性稅主宰：

1. **固定重複稅**：每個生成子代理都重載一次規範＋指令。數萬題若每批數十題，就是上千個代理，每代理重載規範的固定稅乘上千倍。
2. **紅隊輸出量**：一刀切的長 reason 欄乘上數萬則。

核心洞察（本 ADR 的靈魂）：在千批規模，**省 token 與升品質是雙贏、不是取捨**——省下的重複稅可以換成更高的抽樣率、更足的錨定素材。但這只有在「省的是重複稅與輸出格式、不是紅隊本身」時才成立。

## 決策（Decision）

**先立三層分工（貫穿全節，另見 ADR-0008）**：**結構 lint 抓格式、檢索錨定防事實錯（讓模型抄現行條文而非憑記憶）、紅隊抓剩下的事實錯**。token 工程省的是前兩層零 token 手段能接手的部分，**絕不弱化紅隊**——紅隊必跑、必抽，抽樣率只增不因省 token 而減。

**兩個總原則**：（一）省 token＝升品質是雙贏；（二）**抽樣率是品質參數、輸出格式是成本參數，分開調**。

**七槓桿：**

*共通（新寫線／重構線皆適用）*

1. **恆定前綴吃 prompt cache**：每個生成 agent 的 prompt 拆「恆定前綴＋變動尾」。前綴＝蒸餾操作卡＋1 則**已過紅隊**的同科同題型金樣＋科級錨定素材包；前綴須**逐位元組相同**才吃得到 cache，批次題目只放變動尾。配套紀律：**同科波次連續跑、不跨科交錯**（一跨科前綴就變、cache 失效）。素材包設上限（如 10–20k token，按紅隊錯誤統計選常引法規優先收錄；無上限的全文包會反噬 cache 經濟）。錨定素材包的樣子，見 [`examples/pipeline-templates.md`](../../examples/pipeline-templates.md) 的「錨定素材包片段」。
2. **蒸餾操作卡取代整份規範進 prompt**：生成 agent 只吃一張 300–500 token 的**蒸餾操作卡**（三段結構模板＋硬規則）＋1 則金樣；金樣塞 3–5 則反而稀釋遵循率，取一則最相近即可。規範全文只給紅隊裁決者與 lint 開發。配套：**操作卡×lint 覆蓋矩陣**——每一硬條款必須被「操作卡」或「lint 檢查項」至少其一覆蓋，防蒸餾時漏掉 lint 驗不到的內容規則。一張操作卡的完整樣子，見 [`examples/pipeline-templates.md`](../../examples/pipeline-templates.md) 的「操作卡範例」。
3. **lint 由主迴圈直接跑，不開 lint agent**：生成 workflow 只「生成→落 draft 檔」，回主迴圈用一條 shell 對整波 draft 跑結構 lint，主迴圈只看 pass/fail 計數。千批規模下替 lint 另開 agent 的殼開銷可達數十萬 token，純浪費。
4. **紅隊 schema 化＋批次化**：fresh-context 的意義是**與撰寫者隔離**，不是一題一 agent。同一紅隊 call 批 10–20 題（題目彼此獨立），以 structured output 強制輸出並**依判決分層**：`verdict=ok` **零 reason**（九成題在此省最多）；`verdict=fix` 給 `error_type` 枚舉＋`detail≤100字`。一刀切「reason≤30字」會餓死裁決者與回饋圈兩個下游——故分層而非一刀切。**這是壓輸出格式（成本參數）、不動抽樣率（品質參數）。**

*重構線專屬*

5. **內容守恆檢查（零 token）**：抽舊詳解的**關鍵 token 集**（數字、法條號／函釋字號、專有名詞、各選項字母的結論詞），驗證重構後全數保留——防模型重排時默默改寫已驗證推理（事實漂移）。**管線順序（防自相殘殺）**：腳本預清理（剝免責句＋數字指稱→A–E）**先跑、守恆基線建在預清理後的舊詳解上**，token 集排除選項編號數字並做數字正規化（全半形／中文數字），否則合法刪除全被誤殺；每道退回門重試上限（如 2 次）後轉人工佇列。
6. **格式修不重審**：紅隊已過、僅 lint 掛掉的則（純格式），修完只回 lint＋守恆檢查、不再進紅隊。**定義收窄**：格式修＝**不增刪內容 token 的修復**（段落重排、空行、冒號、字母標記）；lint 掛「字數下限」「缺辨析」屬**內容缺陷**，回內容流＋照常紅隊。純格式修下放便宜模型時，兜底為**雙向守恆**（舊 token 全保留＋新增內容 token 比例護欄——單向守恆驗不到「為過門自行補內容」）。

*新寫線專屬*

7. **指紋聚類、一份共用**：新寫池啟動前用內容指紋（題幹＋選項＋答案，**絕不 stem-only**——同題幹異選項是真不同題；**且須納入題組 passage**——題組成員題引用情境脈絡、不可與獨立同文題共用）做零 token 聚類。同科跨年完全相同題只寫一份、整簇套用。跨科重疊常趨近 0，重複多發生在同科跨年。**先量重複率再決定**投入。

## 考慮過的替代方案（Alternatives）

- **A. 每代理載入完整規範＋小批**：最直覺，但千批規模下固定重複稅乘上千倍，且不吃 prompt cache。已被槓桿 1／2 取代。
- **B. 紅隊一題一 agent 求「最純 fresh context」**：誤解了 fresh-context 的意義——隔離的對象是「撰寫者思路」，不是「同批其他題」。題目彼此獨立時批 10–20 題不損隔離、大省殼開銷（槓桿 4）。
- **C. 一刀切壓紅隊 reason（≤30 字）省最多輸出**：省過頭——裁決需要 detail 判真錯／誤報、回饋圈需要 error_type 做 tally，一刀切把這兩個下游餓死。改為分層 schema。
- **D. 素材包收錄該科全部法條全文**：錨定最強，但全文包無上限會反噬 cache 經濟（未命中時每批付全額）。改為設上限＋包外引用標低把握度觸發 100% 紅隊。
- **E. 守恆檢查在預清理前建基線**：會把合法刪除（免責句、選項編號）全誤判成事實漂移，誤殺率爆表。改為預清理先跑、基線建在預清理後。

## 後果（Consequences）

- **省下的 token 直接回灌品質**：抽樣率提高、素材包給足全文讓模型「抄」而非「憑記憶」。這是「省 token＝升品質」的具體兌現。
- **試點批必量三項驗槓桿**：prompt cache 命中率（驗前綴是否逐位元組相同／有無跨科交錯）、內容守恆檢查誤殺率（驗關鍵 token 集抽取沒過嚴）、便宜模型 vs 中階模型在重構線的**分子集** A/B。
- 風險：槓桿本身可能悄悄拿偵測率換 token 而不自知——故「改 batch size 或換 tier 時必重跑摻毒批 recall」（見 ADR-0008）。
- **不能省的（原樣鎖死）**：答案錨 100%、結構 lint 100%、資料不變式、紅隊必跑必抽、敏感科從嚴、新寫線主幹用中階以上模型。省 token 只作用在「重複稅、輸出格式、免費零 token 門」。

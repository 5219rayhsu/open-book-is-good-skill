# ADR-0004：既有舊詳解一律「重構」而非「重寫」

- **狀態**：已採用（Accepted，2026-07-16）
- **日期**：2026-07-16
- **決策者**：專案維護者
- **相關**：`references/explanations-redteam.md` §2.3（重構線專屬槓桿）；ADR-0003（三段結構＝重構的目標格式）；ADR-0005（內容守恆檢查是重構線的兜底門）；ADR-0007（重構線的紅隊抽樣）。

## 背景（Context）

詳解規範升級到三段結構＋考點段（見 ADR-0003）後，既有題庫裡累積的大量「舊版自由散文」詳解對新規範的合規率極低——幾乎全部是舊格式，首段「正解：」、字母段、末段「本題考點：」都近乎零覆蓋。

但盤點也揭示了缺陷的**性質**，這是本決策的關鍵依據：

- **鷹架詞（更正／經查／自我驗證旁白）抽樣近乎零、免責收尾近乎零**：既有正文乾淨。
- **結論正確性多已過紅隊**：達標科的舊詳解帶有紅隊記錄（外部強模型審＋最強模型修）。**正確性是已驗證資產。**
- 真正的硬缺陷只有兩類：(a) **結構**——無三段、無考點段、選項辨析未逐項成段；(b) **過短**——多數只講正解、未逐一辨析誤答，補齊等於新增內容。

問題於是變成：這批舊詳解該「從零重寫」還是「以舊詳解為草稿重構」？

## 決策（Decision）

**決策一：一律「重構」，不從零「重寫」。**

既有詳解的結論與正解理由是**已驗證資產**（已投入紅隊成本、已定案）。從零重寫會重新翻案已定案的答案、浪費既投入的紅隊成本，還可能引入新的結論錯誤。重構以舊詳解為草稿保留該資產，只補結構（三段化）與辨析（逐項誤答、考點段、計算題逐步演算）、修硬傷（數字指稱→A–E、去鷹架詞、去免責收尾）。

- **輸入仍是 clean-room**：題幹＋選項＋官方答案鍵＋舊詳解（當草稿），**不引入出版商詳解**。
- **結論不得翻案**：重構時若發現舊結論可疑（與官方鍵矛盾），標記送最強模型裁決、不逕改；答案錨硬門仍 100% 跑（重構後答案必須＝官方鍵）。

**決策二：工作線分「重構線」（既有舊詳解）＋「新寫線」（真實缺口）。**

- **重構線**＝有堪用舊詳解的科 → 以舊詳解為草稿重構。
- **新寫線**＝真實缺口（從無詳解）→ 一開始就用新規範直接新寫，不經「舊格式→重構」兩道工。
- 同科內：既有先重構、缺口再新寫，同批對帳。

**決策三：分流不在「重構 vs 重寫」，而在紅隊抽樣強度。** 原本過短的科，重構時新增的「選項辨析段＋考點段」是**舊紅隊未涵蓋的淨新內容**，抽樣須提高（詳見 ADR-0007）。

**決策四：重構線先於新寫線。** 重構每則輸入含舊草稿、輸出較短，明顯比新寫（無草稿、輸出較長、金樣 few-shot 較重）便宜且快，先行可先把新結構鋪滿達標科、快速提升全站合規率。

## 考慮過的替代方案（Alternatives）

- **A. 全站從零重寫**：格式最乾淨、無需守恆檢查，但**翻案已定案結論、浪費既投入紅隊成本**，且每則輸出更長更貴，還把「已驗證正確性」這個資產丟掉重賭。已排除。
- **B. 按科分流：內容腐壞的科重寫、只結構缺陷的科重構**：理論上更精準，但盤點證明**沒有科是內容腐壞**（鷹架詞近乎零、正文乾淨、結論已過紅隊），分流前提不成立。真正的差異是「補寫辨析量」，那用抽樣強度處理（決策三）即可，不需要分兩種生成路徑。
- **C. 只補結構、不補辨析**（純機械三段化，過短就過短）：最省，但過短詳解多數只講正解、沒辨析誤答，純三段化等於把空段留白，不符「逐項辨析」硬要求。故過短科的重構必然含淨新增內容。

## 後果（Consequences）

- 保留已驗證正確性、省 token、快速鋪滿新結構——重構明顯比新寫便宜快。
- **新風險：淨新增內容（補寫的辨析）是舊紅隊未涵蓋的**。某國考社工師科的重構線首波實測即印證了這點：真錯率約 10%、錯誤集中在補寫的法律事實層（法條號張冠李戴、法定數字錯、虛構規定、權責用語錯）。這直接催生了 ADR-0005 的內容守恆檢查下探限定與 ADR-0007 的預設全抽。守恆檢查只保「舊內容不漂移」，抓不到「新寫辨析裡的事實錯」，故過短科／法規科不下探抽樣。

## 如何重啟討論／修改（How to revisit）

- 「重構非重寫」的前提是「舊結論已過紅隊＝已驗證資產」。若未來某科的舊詳解**未經紅隊**（如純機械生成、無紅隊記錄），該科不適用本決策、應走新寫路徑並全抽紅隊。
- 分線名單與各科優先序隨題庫變動，屬運營層、不在本 ADR；本 ADR 記的是「為何重構、為何分線」的決策邏輯。

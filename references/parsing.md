# 試題解析（judgment 式，不是死腳本）

把官方歷年 PDF 試題轉成結構化題庫 `bank.json` 的方法論。核心立場：**這是一套判斷式流程，不是一支萬用腳本**。每一卷的版面、題型、答案鍵格式都不同，解析器要先「偵測這卷的實況」，再選對應策略，而不是硬套固定規則。

授權 MIT。試題本身依著作權法 §9，憲法、法律、命令與公務員職務上草擬之考試試題不受著作權保護，故可逐字重製。

## 1. 為什麼用 markitdown 先轉 md

直接用 pymupdf 抽字，拿到的是「座標散落的文字行」，閱讀順序常常壞掉——尤其題組共用的引文（passage）會跟題目交錯、雙欄選項會左右串行。所以第一步一律先用 markitdown 把 PDF 轉成 markdown：

```bash
markitdown data/raw/{年}_{科}_試題.pdf -o data/raw/{年}_{科}_試題.md
```

markitdown 的好處是它有版面理解，散文段落（題幹、題組引文）的閱讀順序大致正確，題組的「甲乙丙」引文會排在標頭之後，這是 raw pymupdf 做不到的。

**但 markitdown 不是萬靈丹**，它會在幾種情況翻車，這時改用 pymupdf 的座標序（`get_text("dict")` 依 `(page, y, x)` 排序）當輔助或主力。判斷哪邊可靠，正是 judgment 的一部分：

- **散文（passage、題幹）** → 信 markitdown，閱讀順序乾淨。
- **多欄並排的選項**（cloze 選項格、會考雙欄 A|B / C|D）→ markitdown 線性化順序各年不一致，**改用 pymupdf 座標 band 分群**才對得齊。
- **複雜統計表**（社會卷）→ markitdown 會把表格塞進 markdown table，需攤平還原；嚴重破碎時用 pymupdf 座標重抓題幹與選項。
- **整批以座標序重建**（會考英語）→ 該卷「題號獨立成行、題幹/選項後移」的版面 markitdown 會張冠李戴，乾脆全卷以 pymupdf `get_text` dict 依 `(page, y, x)` 忠實重建，逐字不漏。

換句話說：markitdown 解決「閱讀序與題組綁定」，pymupdf 補「markitdown 線性化失敗的版面」。兩者互補，由解析器當下判斷用哪個。

## 2. judgment 式：先偵測實況，再選策略

寫解析器前（其實是寫的過程中反覆驗證），先回答這幾個關於「這卷長怎樣」的問題，答案決定策略：

| 偵測項 | 學測（GSAT） | 會考（CAP） |
| --- | --- | --- |
| 有沒有題組？ | 有，且大量（社會幾乎全題組） | 有，後半段固定為題組 |
| 題組標頭長怎樣？ | 「N 至 M 為題組」「第 N 至 M 題為題組」 | 「請閱讀以下…回答 A～B 題：」「閱讀…回答第 a 至 b 題：」、英語用「(N-M)」 |
| 選項是 4 選 1 還 5 選項？ | 國綜/社會 (A)–(E) 五選項、英文 (A)–(D)，詞庫到 (J) | 全部 (A)–(D) 四選一 |
| 有沒有多選題？ | 有（社會、英文混合題；答案多字母） | 沒有，全單選 |
| 有沒有非選/混合題？ | 有（國綜混合題、英文中譯英+作文、社會非選） | 沒有 |
| 答案鍵格式？ | 各科獨立答案 PDF，橫排「題號/答案」表 | 一份多科一覽表 PDF，要定位該科欄位 |
| 圖多不多？ | 社會、數理多；國綜/英文閱讀少 | 社會多；國文少 |

**結論：學測較複雜（5 選項、多選、非選、混合題並存，且每科結構差很多），會考較單純（全 4 選一單選）。** 所以學測幾乎每科一個 parser，會考也每科一個，但會考的判定邏輯相對乾淨。

### 偵測落實到程式的幾個具體手法

- **題型由答案推斷，不靠版面猜**：拿到該題官方答案後，`len(answer) > 1` → 多選；單字母 → 單選；答案是「／」或空且無選項 → 非選。這比從題幹文字猜「這題是不是多選」可靠得多。
- **答案表是權威題號清單**：官方答案 PDF 列出的題號就是這卷的合法題號。用它過濾掉卷末附錄、評分參考表裡被誤判成題號的孤立數字（社會、會考都靠這招）。
- **題組跨度合理性檢查**：題組標頭抓到的 `(a, b)` 要求 `a <= b`、跨度不過大（如會考國文限 `b-a <= 5`、英語 `<= 9`），擋掉誤判。
- **選項殘缺 → 觸發 fallback**：答案明明是 A–E 某字母，但 markitdown 只解析到 < 4 個選項 → 判定版面被打碎，改用 pymupdf 座標重抓該題。

## 3. 忠實原則（紅線）

- **逐字忠實擷取原文**，絕不改寫、摘要、翻譯、抄出版商的編排或詳解。英文題保留英文原文。
- **跳過卷首測驗說明**（約 1.5 頁的注意事項/作答方式/計分說明）。學測從「第壹部分」起算；會考從「一、單題」標頭起算，其前全部丟棄。
- **去頁眉頁腳裝飾行**：頁碼、「第 N 頁/共 N 頁」、「請翻頁繼續作答」、「背面尚有試題」、科目標題、財團法人字樣等，逐行用 `DROP` 正則過濾。
- **題組 group_id / passage 綁定**：同一題組的所有子題共用 `group_id` 與 `passage`（題組共用素材）。passage = 題組標頭之後、到第一個子題題號之前的文字。**題組絕不拆散**（自驗會檢查同 gid 題號連續且符合標頭範圍）。
- **答案表用 find_tables 對應**：答案一律從官方答案 PDF 解析，`pymupdf find_tables` 抓表、逐列對「題號 / 答案」。學測各科一張獨立答案表；會考是多科一覽表，要先定位該科欄位（依表頭含「國文」「社會」「英語/閱讀」字樣動態抓欄 index）。

## 4. 每科一個 parser，但共用 schema 與冪等 merge

各卷結構差太多，硬寫成一支通用 parser 只會變成滿是 `if 科目 == …` 的巨獸。所以**每科各自一個 parser**：

| 腳本 | 對象 | 主要策略特點 |
| --- | --- | --- |
| `parse_gsat.py` | 學測國綜 | markitdown 散文閱讀序 + find_tables 答案；題組「N-M 為題組」 |
| `parse_eng.py` | 學測英文 | 七大 section（詞彙/綜合測驗/文意選填/篇章結構/閱讀/混合/非選）各自切法；cloze 與詞彙選項用 pymupdf 座標 band；詞庫（A–J / A–E）自動偵測尾端連續選項列，不硬編 max |
| `parse_soc.py` | 學測社會 | 攤平 markdown 表格 + 題組標頭三形態容錯（同行/顛倒兩行/正常兩行）；破碎統計表用座標 fallback；領域（歷史/地理/公民）關鍵詞推斷 |
| `parse_cap_國文.py` | 會考國文 | 全 42 題四選一；題組標頭錨定「回答 A～B 題：」；選項容許括號內空白「(  B  )」；清 `(cid:NN)` 與 PUA 殘碼 |
| `parse_cap_社會.py` | 會考社會 | 約 54 題四選一；攤平表格；雙欄選項座標 fallback；千分位大數判定為圖表座標軸殘留 → 視為空選項 |
| `parse_cap_英語.py` | 會考英語（閱讀） | **整卷以 pymupdf 座標序重建**；字彙註解（英文+中文釋義）整組剝除；頁碼洩漏（選項尾端裸數字）清除 |

它們**共用同一份題目 schema**（見 §6）與**冪等的 qid-keyed merge**：

```python
def merge_bank(new_subject):
    obj = json.load(open(BANK)) if exists else {"meta": {}, "questions": []}
    kept = [q for q in obj["questions"] if q.get("subject") != SUBJECT]  # 只移除本科舊題
    merged = kept + new_subject                                          # 換上本科新題
    obj["questions"] = merged
    json.dump(obj, open(BANK, "w"), ensure_ascii=False, indent=1)
```

關鍵：merge 以 `subject`（或更細到 `qid`）為鍵，**重跑某科只覆蓋該科、不動其他科**。所以 `parse_eng.py` 重跑不會破壞 `parse_gsat.py` 的成果。學測各科直接 merge 進 `data/bank.json`；會考各科先各自輸出到 `data/_stage/cap_{科}.json`，合併由上游（`merge_stage3.py`）負責，同樣 qid-keyed 冪等。

`qid` 命名保證跨年跨科唯一：
- 學測：`{year}_{科}_{no}`（如 `114_社會_31`）；英文非選用 `{year}_英文_trans1`、`_essay` 避免與選擇題 no 衝突。
- 會考：`{exam}_{year}_{科}_{no}`（如 `會考_114_社會_31`）。

## 5. 圖片：只存檔名，build 時才內嵌（絕不在題庫存影像）

圖無法以文字忠實重建，所以解析階段**只標旗標、不存影像**：

1. **解析器標 `needs_figure`**：題幹/passage 出現「圖/表/地圖/附圖/照片/示意圖/統計圖…」等指涉詞，或選項數 < 4（圖被吃掉）、或選項全空（純圖選項）→ `True`。社會卷圖多，原則「寧可多標」。
2. **`extract_figures.py` 把需要圖的題 render 成獨立 PNG**：對 `needs_figure==True` 的題，定位其 stem 在原卷 PDF 的版面位置，取「該題頂端到下一題頂端」的版面區塊（band），`page.get_pixmap(clip=...)` render 成乾淨 PNG 存進 `data/figures/`。命名：獨立題 `{簡碼}{year}_q{no}.png`、題組 `{簡碼}{year}_g{首題}_{末題}.png`（一組共用一張）。**bank.json 只寫入 `figure` 欄的檔名字串，永不寫影像。**
3. **`build_app.py` 在 build 時才內嵌**：`figures_block` 讀 `data/figures/` 下 PNG，編成 `window.__FIGS__ = {檔名: dataURI}` 注入單檔 HTML，前端據 `figure` 欄查表顯示。

這條資料流是刻意設計的：

```
原卷 PDF → render → data/figures/*.png（二進位）
                         │
bank.json.figure = "S114_q24.png"（只有檔名，純文字、可 lint、可進 git diff）
                         │
build 時 build_app.py 讀 PNG → 內嵌進 window.__FIGS__ → 單檔離線 HTML
```

**鐵則：題庫（bank.json）永遠是純文字。** 影像只在 `data/figures/` 與最終 HTML 裡存在；編碼字串只在 build 函式內部與輸出 HTML 中流動，**任何階段都不得把編碼字串 print 出來或寫進題庫**（會觸發內容過濾器、也讓 bank.json 無法被人讀與 lint）。

## 6. bank.json 題目 schema（每題）

```jsonc
{
  "qid":          "114_社會_31",   // 跨年跨科唯一鍵；merge 與 figure 對應都靠它
  "exam":         "學測",          // 學測 / 會考
  "year":         114,
  "subject":      "社會",
  "no":           31,              // 卷內題號（非選用 null，qid 帶 trans1/essay）
  "group_id":     "114_社會_g31_32", // 題組鍵；獨立題為 null
  "passage":      "……題組共用素材……", // 題組共用引文/選文/材料；獨立題為 ""
  "stem":         "……題幹……",
  "options":      { "A": "…", "B": "…", "C": "…", "D": "…" }, // 詞庫題存共用詞庫；純圖選項可空
  "answer":       "A",            // 可多選（多字母如 "AC"）；非選為 ""
  "type":         "單選",          // 單選 / 多選 / 題組 / 非選
  "needs_figure": false,          // 是否需對照圖
  "figure":       null            // extract_figures 寫入的 PNG 檔名；無圖為 null
}
```

各科另可帶選用欄位（不影響共用流程）：學測社會的 `domain`（歷史/地理/公民推斷）、英文的 `section`（詞彙/綜合測驗/…）。`meta` 區塊記來源、年度、parser 說明與題數，供稽核。

## 7. 自驗：把「忠實」變成可檢查的條件

每個 parser（尤其會考三科）跑完都做逐年自驗，只回報**真正的異常**，不把圖片題的合理空缺當錯：

- 題數對不對（會考國文 42、社會 54；英語各年不一但題號要連續）。
- 題號連續、無重複、無缺漏；答案皆在合法集合（A–D / A–E）。
- 選項齊全：**非圖片題**才要求 4 個選項齊全；`needs_figure` 的圖片題容許選項殘缺（選項本身是圖）。
- 無選項洩漏：某選項內容不應殘留其他選項標記 `(X)` 或下一題題號。
- 題組未拆散：同 `group_id` 題號連續且與標頭宣告的 `a..b` 一致；文字選文題組 passage 須非空且組內一致，圖片選文題組容許空 passage。

自驗的精神跟整份方法論一致：**規則要分情況（圖片題 vs 文字題、題組 vs 獨立題），不一刀切**——這正是 judgment 式解析的縮影。

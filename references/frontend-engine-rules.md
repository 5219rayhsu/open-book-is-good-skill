# 前端引擎規則：題組情境、出題演算法、標記與儲存

> 適用 Phase 4（`bank.json` 的題組欄位：`group_id`／`passage`／`standalone`）與 build 完成後的
> **前端作答引擎**本身（單題練習、模擬考、錯題重練這類互動邏輯，不是打包腳本）。
>
> **為何獨立成檔、不併入 `references/build-deploy.md`**：build-deploy 講的是「程式碼／資料 → 給人
> 用的成品」這一哩——打包單檔、PWA、體積控制，關注的是**產物交付**。本檔講的是**引擎行為**本身：
> 使用者在不同作答表面上，題組情境怎麼顯示、下一題怎麼被演算法選出、標記與儲存兩個相似功能怎麼
> 分工。兩者關注點不同（交付形態 vs 互動邏輯），硬併會讓 build-deploy 從「打包手冊」膨脹成「打包
> ＋引擎行為」的大雜燴，故獨立一檔。

## 1. 題組／承上題的情境引入規則

題庫裡有兩種「不可單獨理解」的題：**資料層題組**（`bank.json` 有 `group_id` 與共用 `passage`）
與**承上題**（題幹以「承上題」等詞開頭，情境在前一題題幹裡，資料層沒有連結欄位）。網站的多個作答
表面——歷屆原卷、模擬考、完整診斷、單題練習、錯題重練、弱點殲滅——都可能以**非原卷順序**遇到
這兩種題（混卷抽題、跨模式重練、單題拆出練習），「情境要不要補、補多少、補在哪」因此需要一份
跨表面一致的規則，不能每個表面各自土法煉鋼。

### 核心決策（四條）

1. **原文完整引入，禁止抽樣與摘要。** 情境框顯示錨題「題幹全文」，唯一允許的處理是純版面折行
   重組（不動內容）。理由三點：連環題的作答關鍵常是錨題的單一數字或條件，節錄等於賭哪句不重要；
   摘要會引入 AI 改寫層，違反「題庫忠於官方原文」的底線；依著作權法第 9 條，考試試題全文引用
   零法律成本。
2. **只引題幹，不引選項與正解。** 選項與答案不是情境載體，引入只會膨脹版面，還可能剝奪使用者
   之後獨立練到錨題的機會。
3. **鏈式堆疊。** 連環承上（例：第 33→34→35 題）顯示「錨題＋沿途每個中間承上題」的題幹，依原卷
   題號升冪、各標「第 N 題」——只顯示緊鄰前一題，會漏掉連環中間題的情境。
4. **模式分工，不強求所有作答表面統一處理**：

   | 作答表面 | 情境處理 |
   | --- | --- |
   | 歷屆原卷 | 自然順序、零情境框（相鄰抑制：畫面上一題即錨題時不重複顯示） |
   | 模擬考／完整診斷 | 抽題後做「鏈相鄰正規化」（同鏈成員聚攏到首個被抽中位置、依原題號升冪）＋「錨題拉入」（承上題被抽中而錨題落選時，換入同科一題非題組題頂替錨題，換不進則退回情境框） |
   | 單題練習／錯題重練／弱點殲滅／歷史詳情 | 情境框自足、不聚攏（逐題節奏優先，每題須可獨立作答） |

   「歷屆原卷聚攏」會打亂真實卷序、「模擬考不聚攏」會讓使用者頻繁抽到承上題卻看不到情境——兩種
   極端都被排除，才落成上表這種依表面分流的做法。

### 觸發與解析判準

資料層（`group_id`／`passage`）優先於 runtime 回溯；兩者都沒有，且題幹命中固定詞表（承上題、依
前文、依上文、承前題、同上題、依前題、根據上題、接上題）才走承上回溯——在同一卷內往前找第一個
非承上題當錨；找不到卷或錨題時顯示誠實 fallback 文案，引導使用者去「歷屆原卷」看整卷。

**詞表是正典**：新增觸發詞（官方考卷用語千變萬化，詞表不可能一次窮舉）必須同步更新到你的設計
決策紀錄裡，別讓詞表與文件分家。

> **為什麼記這麼細**：上面四條決策各自都有一個「更省事但更差」的替代方案——摘要引入、情境框帶
> 選項、只看緊鄰前一題、所有表面統一處理——都在實測後被排除。復刻本手冊時若圖省事走了被排除的
> 替代方案，八成會重踩同樣的坑。建議把完整決策脈絡、排除掉的替代方案與理由、可復刻的驗證配方
> 另存一份決策紀錄（ADR），別讓這類「為什麼不這樣做」的理由只存在某次對話或 commit message 裡。

## 2. 出題演算法：弱項加權、指紋冷卻、回鍋複習

單題練習模式（無限、自適應）挑下一題不是純隨機，而是疊了幾層規則，**疊加順序本身就是優先序**：

1. **使用者手動佇列優先**（如錯題本點的「重練此組」），其次是答錯觸發的關聯補強佇列。
2. **回鍋複習（定期插入）**：每答對固定題數（實務上每 5 題）安插一題「練過但還沒精熟」的舊題——
   單純刷新題容易記不牢，回鍋複習鞏固印象。候選限定「有作答紀錄、未達精熟門檻、且不在最近一段
   作答紀錄內」；科目挑選沿用下面第 3 點的弱項加權，避免回鍋複習又是弱項加權之外的另一套邏輯。
3. **弱項加權挑科（Laplace 平滑 ＋ 保底權重）**：科目權重公式如下（保底值實務取 0.15）：

   ```
   weight(subject) = (1 - (ok + 1) / (n + 2)) + 0.15
   ```

   - Laplace 平滑那一項讓「還沒作答過的科目」不會因為分母是 0 就權重無限大——比照
     樣本數很大時的中庸值，避免新科被過度優先或過度冷落。
   - 保底值確保**強科（正確率接近滿分）仍保留一定機率被抽到**，維持手感不生疏，不會因為練得好
     就再也不出現。
4. **跨年重複考古題冷卻**：候選池排除「內容指紋（見 `references/shared-question-banks.md` 的
   內容指紋定義）出現在最近一段作答紀錄（實務上取最近 30 筆）」的題——同一份考古題換年份換
   題號重複收錄，剛練過的內容不該馬上又抽到。**排除後若候選池空了要退回原本候選**，避免題池被
   冷卻規則榨乾而卡死。**短題幹護欄同樣適用於這裡**：過短的題幹（合法的詞彙型單題）不受冷卻
   排除。
5. **兜底**：全題庫都練過時，從全池挑「精熟複習次數最低的一批」隨機出一題，維持「哪裡最生疏就
   多練哪裡」的一貫精神。

## 3. 標記與儲存：兩個相似功能，職責不同

作答卡片上常見兩顆按鈕，容易被誤以為是同一件事，實則職責分開：

- **標記（flag）**：純旗標，與答對答錯無關，使用者自己覺得「這題要注意」就點一下。呈現方式是
  **作答內記號**（當下卡片標記狀態變化，可切換）＋ **回顧介面的視覺提示**（錯題回顧、歷史列表
  右上角一個小折角），讓使用者掃過清單就認出哪些題自己標過。
- **儲存（save）**：把題目收進一個獨立的「儲存專練」集合，供之後開一個**獨立頁面只練這個集合**
  ——這是建立一份子題庫，不是單純的旗標。

兩者都是 qid 對應簡單值的映射（標記存布林、儲存存時間戳），**但語意不同就不該共用一套資料結構
或一顆按鈕**：標記解決「回顧時認得出來」，儲存解決「之後只練這批」。少了這個區分，使用者會把
「這題我想再看一眼」跟「我要把這些題整理成一份專練」混在一起，功能退化成沒人用得順手的四不像。
